🔐 Authentication

Public GET endpoints ไม่ต้องใช้ token — Admin write endpoints ต้องมี Firebase ID Token

ℹ️
Admin verification ตรวจสอบสิทธิ์ 2 วิธีตามลำดับ:
1. Firebase Custom Claims — ถ้า token มี claim admin: true
2. Environment Variable — ถ้าอีเมลอยู่ใน ADMIN_EMAILS (.env)
Header Format
HTTP Header 
Authorization: Bearer <firebase-id-token>
Auth Error Responses
401 ไม่พบ token หรือ token ไม่ถูกต้อง — "ไม่พบ token การยืนยันตัวตน"
403 Token ถูกต้องแต่ไม่มีสิทธิ์แอดมิน — "ไม่มีสิทธิ์แอดมิน"

⚠️ Error Handling

ทุก error response มีรูปแบบ JSON เดียวกัน — ข้อมูลภายในไม่รั่วไหลออกสู่ client

Error Response Format
JSON 
{
  "message": "คำอธิบายข้อผิดพลาด (ภาษาไทย)"
}
HTTP Status Codes
200 สำเร็จ — ดึง/แก้ไข/ลบข้อมูลเรียบร้อย
201 สร้างข้อมูลใหม่สำเร็จ
400 ข้อมูลไม่ถูกต้อง — validation error (category, id, slug, หรือ body)
401 ไม่มี token หรือ token ไม่ถูกต้อง
403 ไม่มีสิทธิ์แอดมิน
404 ไม่พบข้อมูลที่ร้องขอ
429 คำขอมากเกินไป (rate limit exceeded)
500 เกิดข้อผิดพลาดภายในระบบ (generic message, ไม่เปิดเผยรายละเอียด)

✅ Validation Rules

กฎการตรวจสอบข้อมูลที่ backend ใช้ก่อนเขียนลง Firestore

Document ID
  • Pattern: /^[a-zA-Z0-9_-]+$/
  • ความยาวสูงสุด: 128 ตัวอักษร
  • ต้องไม่เป็นค่าว่าง
Law Categories
computer privacy copyright
Card Categories
"" (empty) help article resource
Slug
  • Pattern: /^[a-z0-9]+(?:[-_][a-z0-9]+)*$/
  • ถูก normalize เป็น lowercase อัตโนมัติ
  • ความยาวสูงสุด: 100 ตัวอักษร
URL Fields (imageUrl)
  • ต้องเป็น valid URL
  • อนุญาตเฉพาะ protocol http: และ https:
  • ความยาวสูงสุด: 2,048 ตัวอักษร
HTML Content (pageContent)
  • ความยาวสูงสุด: 20,000 ตัวอักษร
  • Allowed tags: p, br, strong, b, em, i, u, ul, ol, li, h2, h3, h4, blockquote, a, img
  • Tags อื่นๆ เช่น script, style, iframe จะถูกลบออก
  • Event attributes เช่น onclick, onerror จะถูกลบ
  • Link (a href) อนุญาตเฉพาะ http:, https:, mailto:, tel:
  • javascript: URLs ถูก reject
  • Attribute class ถูก sanitize ให้เหลือเฉพาะ /^[a-zA-Z0-9_-]{1,50}$/ สูงสุด 8 classes

POST /api/cards Admin

สร้าง card ใหม่ — ระบบจะเพิ่ม createdAt และ updatedAt อัตโนมัติ เนื้อหา pageContent จะถูก sanitize ก่อนเก็บ

Request Body
FieldTypeRequiredDescription
title string Required ชื่อหัวข้อ — สูงสุด 200 ตัวอักษร
subtitle string Optional หัวข้อย่อย — สูงสุด 300 ตัวอักษร
description string Optional รายละเอียด — สูงสุด 1,000 ตัวอักษร
imageUrl string Optional URL รูปภาพ — http/https เท่านั้น, สูงสุด 2,048 ตัวอักษร
slug string Optional URL-friendly identifier — lowercase, a-z 0-9 _ - เท่านั้น
pageContent string Optional เนื้อหา HTML ที่ถูก sanitize — สูงสุด 20,000 ตัวอักษร
category string Optional ค่าที่อนุญาต: "", help, article, resource

📦 Data Schemas

โครงสร้างข้อมูลหลักที่ API ใช้งาน

Card Object
TypeScript-style Schema 
// Card — บทความ/ข้อมูลศูนย์ช่วยเหลือ
{
  id:          string    // Firestore doc ID (auto-generated)
  title:       string    // required, max 200
  subtitle:    string    // optional, max 300
  description: string    // optional, max 1,000
  imageUrl:    string    // optional, http/https only, max 2,048
  slug:        string    // optional, lowercase a-z0-9_-, max 100
  pageContent: string    // optional, sanitized HTML, max 20,000
  category:    string    // "" | "help" | "article" | "resource"
  createdAt:   DateTime  // auto-set on create
  updatedAt:   DateTime  // auto-set on create & update
}
Law Object
TypeScript-style Schema 
// Law — รายการกฎหมาย IT
{
  id:          string    // Firestore doc ID
  section:     string    // required, max 80 (เช่น "มาตรา 5")
  title:       string    // required, max 200
  description: string    // required, max 5,000
  penalty:     string    // required for computer & copyright, max 2,000
                          // optional/absent for privacy
}

🔒 Security

มาตรการรักษาความปลอดภัยที่ backend ใช้งาน

HTTP Headers
Helmet.js
CORS
Origin Allowlist

100 req / 15min

Security Features
  • Frontend ไม่เข้าถึง Firestore โดยตรง — ผ่าน backend API เท่านั้น
  • Firestore rules ปิดกั้น client reads/writes
  • Backend ใช้ Firebase Admin SDK
  • Write endpoints ต้องมี admin auth
  • CORS กำหนด allowed origins ผ่าน ALLOWED_ORIGINS env
  • Production ต้องตั้ง ALLOWED_ORIGINS มิฉะนั้น server ไม่ start
  • Helmet.js ตั้ง security headers อัตโนมัติ (CSP, XSS protection, etc.)
  • Request body ถูกจำกัดขนาด 10kb เพื่อป้องกัน abuse
  • User content ถูก sanitize ทั้งตอนเขียนและอ่าน
  • Error responses ไม่เปิดเผยข้อมูลภายใน (stack trace, file path)
  • ไฟล์ .env และ service account keys อยู่ใน .gitignore
🚨
XSS Protection: ข้อมูลจาก user ถูก sanitize ทั้ง input (เขียน) และ output (อ่าน) — tags ที่ไม่อนุญาตถูกลบ, event handlers ถูกลบ, javascript: URLs ถูก reject
